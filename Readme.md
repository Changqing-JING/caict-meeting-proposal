# Warpo & Warp —— 支持高频动态更新的嵌入式轻量级编译工具链

## 项目概述

Warpo 和 Warp 是一套开源的 WebAssembly（Wasm）编译工具链，专为**资源受限且需要高频动态更新的嵌入式环境**设计。工具链分为两层：

- **Warpo**：类 TypeScript 高级语言 → Wasm 字节码编译器
- **Warp**：Wasm 字节码 → 原生机器码编译器 + 运行时

两者配合，形成从高级脚本语言到嵌入式原生执行的完整链路。

## 要解决的问题

在物联网和边缘计算场景中，设备端常需要**动态下发和执行业务逻辑**。传统方案要么依赖解释型脚本语言（性能差），要么依赖重量级运行时（资源占用高），且往往无法覆盖多种硬件架构，导致技术栈碎片化、重复建设严重。

我们需要一套统一的轻量方案：**一次编写，多端执行，接近原生性能，同时保证安全隔离。** 相比传统 OTA，它更适合高频、小粒度业务逻辑的快速交付、热更新与即时撤回。

典型应用场景：在嵌入式设备上运行轻量级无 UI 应用，以更低成本支持业务逻辑的敏捷迭代。

## Warpo —— 高级语言到 Wasm 的编译器

**仓库**：[wasm-ecosystem/warpo](https://github.com/wasm-ecosystem/warpo)｜**协议**：Apache-2.0

### 解决的痛点

市面上能编译到 Wasm 的脚本语言选择有限。C/C++、Rust 等系统语言学习门槛高，不适合非专业开发者。AssemblyScript 虽然语法接近 TypeScript，但社区维护停滞，缺少闭包、GC、调试器等关键功能，且编译器架构限制了进一步优化的空间。

### 技术优势

- **基于 AssemblyScript 重构**：保留了 TypeScript 风格的语法（上手门槛低），同时重新设计了编译器架构，引入中间表示（IR）层，为优化提供了基础
- **补齐关键语言特性**：支持闭包、垃圾回收优化、调试符号等 AssemblyScript 缺失的重要功能
- **多层编译优化**：函数内联、指令简化、冗余加载消除、热点全局变量提升等优化 Pass，显著提升生成代码质量

## Warp —— Wasm 到原生机器码的编译器

**仓库**：[wasm-ecosystem/wasm-compiler](https://github.com/wasm-ecosystem/wasm-compiler)｜**协议**：Apache-2.0

Warp 的核心编译技术已发表学术论文：Scheidl, F. *"Valent-Blocks: Scalable High-Performance Compilation of WebAssembly Bytecode For Embedded Systems"* (2020)。

### 解决的痛点

主流 Wasm 运行时（如 V8、Wasmtime）体积大、依赖多，无法运行在裸机或资源极度受限的嵌入式系统上，也不支持部分专用指令集架构。同时，汽车/工业等安全关键领域对代码质量有严格的合规要求，通用开源运行时难以满足。

### 技术优势

- **极致轻量**：编译器 + 运行时仅约 187 KB，零内部动态内存分配
- **高性能**：单遍编译，编译速度与执行性能均接近原生水平
- **多架构**：支持 x86_64、AArch64、TriCore，覆盖服务器到微控制器
- **安全沙箱**：内存边界检查、栈溢出保护、异常捕获，隔离宿主系统
- **跨平台**：Linux、QNX、Windows、MacOs、裸机均可运行
- **合规友好**：遵循 MISRA C++ / AUTOSAR C++14 编码规范

## 项目现状与开源目标

两个项目自 2022 年起已在全球量产车辆中持续运行，经过长期、大规模生产环境验证。现采用 Apache-2.0 协议开源，我们希望通过开源：

1. 让更多嵌入式和边缘计算开发者受益于轻量级 Wasm 工具链
2. 与高校和社区合作，推进编译器测试方法论研究，提升编译器可靠性
3. 丰富 Wasm 在嵌入式领域的生态
